ROCm/ROCm — internal architecture

How the docs site and release tooling fit together

This repository doesn’t hold ROCm’s source — it holds the documentation site, the changelog/release generator, and legacy build scripts that operate on top of the component repos. Three independent zones share a handful of root-level files.

CI .github/workflows/linting.yml delegates to the reusable ROCm/rocm-docs-core workflow — markdownlint-cli2 (.markdownlint-cli2.yaml) and spellcheck (.spellcheck.local.yaml) run across docs/ and root Markdown on every push/PR.
Component source (rocm-systems, rocm-libraries, llvm-project, TheRock, and others) lives in separate repositories. This repo only documents and packages releases for them — see CLAUDE.md for the full contributor-facing guide.
